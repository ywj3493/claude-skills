#### 주요 흐름 (Main Flow)

| # | 액터 | 시스템 |
|---|------|--------|
| 1 | 회원가입 페이지에 접근한다 | 회원가입 폼을 표시한다 |
| 2 | 이메일, 비밀번호를 입력한다 | 실시간 유효성 검사를 수행한다 |
| 3 | "가입" 버튼을 클릭한다 | POST /auth/signup 요청을 보낸다 |
| 4 | | 성공 응답을 받으면 로그인 페이지로 리다이렉트한다 |

#### 대체 흐름 (Alternative Flow)

- **A1**: 소셜 계정으로 가입
  1. 사용자가 소셜 로그인 버튼을 클릭한다
  2. 시스템이 OAuth 제공자 페이지로 리다이렉트한다
  3. 인증 완료 후 콜백 처리 및 계정 생성

#### 예외 흐름 (Exception Flow)

- **E1**: 이미 가입된 이메일
  1. 시스템이 "이미 가입된 이메일입니다" 메시지를 표시한다
  2. 로그인 페이지 링크를 제공한다
- **E2**: 네트워크 오류
  1. 시스템이 "네트워크 오류가 발생했습니다. 다시 시도해 주세요" 메시지를 표시한다
